## ⚙️ How It Works

This software uses an AI swarm to perform tasks. Each agent has a specific role, such as price analysis, risk assessment, or transaction execution. When you set a goal, the agents coordinate to find the best way to achieve that result. 

The software utilizes a protocol known as x402. This protocol allows agents to handle their own self-executing wallets. These wallets interact with smart contracts on your behalf. You provide the instructions, and the software handles the blockchain interactions. This method reduces errors that occur during manual trading.

Your funds remain under your control at all times. The software acts as an interface that sends commands to your private keys. It does not store your seed phrases on external servers. All processing occurs locally on your Windows machine. You can view the activity log in the main dashboard at any time to see every action the agents took.

## 📈 Managing Agent Behavior

You control the swarm through the settings panel. You can limit the risk levels for each agent or set a maximum budget for daily operations. If you want to stop all activity, the master kill switch immediately halts every running agent. This gives you full oversight while allowing the software to perform the heavy lifting.

## 💡 Frequently Asked Questions

The software pauses current tasks and resumes them safely once the connection returns. It checks the status of every transaction before continuing.

Do I need to leave my computer on?
Yes, the agents run locally on your hardware. Keep the application open for the agents to continue monitoring the markets. You can minimize it to the system tray to keep your desktop clear.

How do I update the software?
The application checks for updates when you start it. If a new version exists, it will prompt you to download and install the update.

Can I connect multiple wallets?
The framework supports multiple wallet addresses. You can assign different agents to manage different treasuries or portfolios.

Does this work on virtual machines?
Yes, the application runs on standard Windows virtual machines as long as they meet the hardware requirements.
